# 프로젝트 위그드라실

## 개요

프로젝트 위그드라실은 소설, 게임, 기타 창작물을 포함해 다양한 형태로 확장될 수 있는 창작 세계관 설정 프로젝트다.

## 프로젝트 구조

- `룩스테라`: TRPG 팀의 D&D 기반 캠페인에 사용될 배경으로 설계된 판타지 장르 하위 프로젝트다. 현재 가장 적극적으로 제작이 진행된 상태다.
- `엘드로스`: 포스트 아포칼립스 헌터물을 무대로 하는 하위 프로젝트다. 기반 설정이 이미 존재하며, 추후 본격적으로 확장할 예정이다.

## 세계관 관계

- `룩스테라`와 `엘드로스`는 서로 완전히 분리된 독립 세계가 아니라, 기본적으로 같은 세계관 계통 위에 놓인 연결된 프로젝트다.
- 다만 두 프로젝트는 동일한 시공이나 동일한 무대를 공유하는 것이 아니라, 서로 다른 시공과 무대를 가지는 프로젝트로 이해한다.
- 앞으로 추가되는 다른 하위 프로젝트들도 같은 세계관 계통 안에서, 서로 다른 시공 또는 무대를 가진 형태로 이어질 가능성이 높다.

## 현재 기준

- 현 시점에서는 `룩스테라`가 중심 개발 대상이다.
- `엘드로스`는 후속 확장 대상으로 간주하고, 기존 기반 설정을 보존한 채 나중에 내용을 추가한다.

## 시리즈 공통 폴더 구조

각 하위 프로젝트는 되도록 아래 폴더 구조를 공통 기준으로 따른다.

- `비밀 설정`: 아직 공개 전인 설정을 모아두는 폴더다.
- `설정`: 공개된 설정을 정리하는 폴더다.
- `신격`: 공개된 신격 정보를 정리하는 폴더다.
- `템플릿`: 각종 설정 양식을 보관하는 폴더다.
- `설정 정비`: `설정` 및 `신격`을 기반으로, 신규 적용 사항이나 변경 예정 사항을 정리하는 폴더다.
- `스토리 설계`: 이야기 전개와 사건 설계를 다루는 폴더다.
- `세력 관계`: 세력, 인물, 단체 사이의 관계성을 정리하는 폴더다.

템플릿 문서는 `설정_국가 템플릿`, `설정_단체 템플릿`처럼 분류와 대상을 드러내는 방식으로 이름 짓는다.

현재 `룩스테라`의 `템플릿` 폴더에는 아래 템플릿이 존재한다.

- `설정_국가 템플릿`: 국가 설정 작성 양식이다.
- `설정_단체 템플릿`: 단체 설정 작성 양식이다.

## 룩스테라 문서 작성 지침

`룩스테라`에서 새 문서를 만들 때는 아래 메타 항목과 분류 체계를 기본 기준으로 삼는다. 아래 내용은 원래 데이터베이스 구조를 바탕으로 정리한 것이므로, 실제 문서 작성 시에는 문맥에 맞게 유도리 있게 적용한다.

### 비밀 설정 문서

비밀 설정 문서는 아래 메타 항목을 기본으로 사용한다.

- `이름`: 문서 제목이다.
- `비밀 등급`: `하급`, `중급`, `상급`, `최상급`, `태고` 중에서 선택한다.
- `비밀 유형`: `인물`, `물건`, `장소`, `개념`, `사건` 중에서 선택한다.
- `상태`: `시작 전`, `진행 중`, `완료` 중에서 선택한다.
- `생성 일시`: 자동 생성 항목으로 본다.
- `최종 편집 일시`: 자동 갱신 항목으로 본다.

### 설정 문서

설정 문서는 아래 메타 항목을 기본으로 사용한다.

- `이름`: 문서 제목이다.
- `상태`: `시작 전`, `임시작성`, `진행 중`, `완료` 중에서 선택한다.
- `타입`: `종족`, `사건`, `인물`, `아이템`, `지역`, `대륙`, `국가`, `도시국가`, `단체`, `주요 개념`, `개념`, `신격`, `도시`, `지형`, `부족국가` 중에서 선택한다.
- `서브타입`: 필요할 경우 `항구`, `주요`, `일반`, `연맹`, `대륙`, `지침` 중에서 선택한다.
- `영역`: 필요할 경우 `에리디안`, `발가르`, `청연`, `네베아`, `실란도르`, `칼디라`, `녹턴`, `카르코사`, `범대륙`, `진홍의 해협`, `심연의 의회` 중에서 선택한다.
- `생성 일시`: 자동 생성 항목으로 본다.
- `최종 편집 일시`: 자동 갱신 항목으로 본다.

### 설정 정비 문서

설정 정비 문서는 아래 메타 항목을 기본으로 사용한다.

- `이름`: 문서 제목이다.
- `진행 상황`: `준비중`, `작성중`, `작성완료` 중에서 선택한다.
- `작성일자`: 날짜 또는 날짜 범위를 기록한다.
- `최종 편집 일시`: 자동 갱신 항목으로 본다.

### 신격 문서

신격 문서는 아래 메타 항목을 기본으로 사용한다.

- `이름`: 문서 제목이다.
- `대표`: 체크 여부로 표시한다.
- `청연 만신전`: 체크 여부로 표시한다.
- `상태`: `시작 전`, `진행 중`, `완료` 중에서 선택한다.
- `등급`: 현재 확인된 값 기준으로 `반신격`, `소신격`, `중신격`, `대신격`, `주신격`, `초신격` 등을 사용한다. 일부 중간 값은 아직 정리가 덜 된 상태이므로, 필요 시 기존 문맥을 우선 참고해 유도리 있게 적용한다.
- `성향`: 현재 확인된 값 기준으로 `선신`, `회색신`, `악신` 등을 사용한다. 이 역시 일부 값은 추가 확인이 필요하므로 문맥에 맞게 적용한다.
- `도메인`: 다중 선택 항목으로 보고, 현재 확인된 값 기준으로 `빛`, `정화`, `어둠`, `원소`, `생명`, `죽음`, `질서`, `전쟁`, `수호`, `개척`, `지식`, `신비`, `운명`, `행운`, `부패`, `의지`, `평화`, `자연`, `재앙`, `시간`, `광기`, `여행`, `화로`, `예술`, `복수`, `꿈`, `대장간`, `황혼`, `폭풍`, `무덤`, `혼돈`, `기만` 등을 사용한다.
- `포트폴리오`: 현재는 고정 선택지 체계라기보다 사실상 문자열 목록에 가깝게 운용한다. 일부 확인된 값으로는 `천명`, `천계 관료제`, `황권`, `예법`, `천문율`, `왕조의 승인`, `질서`, `역천`, `마도`, `강자존`, `약육강식` 등이 있다.
- `등급 정렬값`: 내부 정렬용 계산 항목으로 본다.
- `성향 정렬값`: 내부 정렬용 계산 항목으로 본다.
- `생성 일시`: 자동 생성 항목으로 본다.
- `최종 편집 일시`: 자동 갱신 항목으로 본다.

### 세력 관계 문서

세력 관계 문서는 아래 메타 항목을 기본으로 사용한다.

- `이름`: 문서 제목이다.
- `관계`: `우호`, `견제`, `대립`, `전쟁` 중에서 선택한다.
- `A(세력)`: 설정 문서와 연결되는 항목으로 본다.
- `A(신격)`: 신격 문서와 연결되는 항목으로 본다.
- `B(세력)`: 설정 문서와 연결되는 항목으로 본다.
- `B(신격)`: 신격 문서와 연결되는 항목으로 본다.
- `쟁점(한 줄)`: 한 줄 요약 텍스트를 적는다.
- `세션 징후`: 세션에서 드러날 단서나 징후를 텍스트로 적는다.
- `공개 수준`: `공개`, `부분 공개`, `비공개` 중에서 선택한다.

### 스토리 설계 문서

스토리 설계 문서는 아래 메타 항목을 기본으로 사용한다.

- `이름`: 문서 제목이다.
- `타입`: `스토리`, `구상` 중에서 선택한다.
- `진행 상황`: `진행중`, `진행완료`, `설계중`, `준비중`, `작성완료` 중에서 선택한다.
- `작성일자`: 날짜 또는 날짜 범위를 기록한다.
- `최종 편집 일시`: 자동 갱신 항목으로 본다.

### 표현 원칙

- 문서 지침에서는 `비밀 설정 DB`, `설정 DB`, `신격 DB`처럼 데이터베이스 중심 표현을 쓰지 않는다.
- 대신 `비밀 설정 문서`, `설정 문서`, `신격 문서`, `세력 관계 문서`, `스토리 설계 문서`처럼 실제 작성 대상 문서 기준으로 표현한다.
- 새 문서를 만들 때는 위 메타 항목을 가능한 한 문서 상단에 드러나게 적어, 분류와 상태를 한눈에 알 수 있도록 한다.

## 구조 적용 기준

- `룩스테라`는 이미 이 구조를 따르고 있으므로, 폴더명은 공통 기준에 맞게 정리해 사용한다.
- `엘드로스`는 아직 본격적인 내용이 비어 있어도, 같은 구조의 폴더 골격은 미리 구성해 둔다.
